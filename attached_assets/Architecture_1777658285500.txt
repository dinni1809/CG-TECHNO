## 1. Architecture Overview

CG Techno Electronics is built on a **Nx Monorepo** architecture using **Next.js 14
with the App Router**. The system is static-first with serverless API routes for form
handling. There is no traditional backend server or database вҖ” all submission logic
is handled at the edge via Next.js API routes and Nodemailer SMTP.

**Core Principles:**
- Static rendering by default (SSG) for all content pages
- Serverless functions only for form processing
- Modular Nx library architecture for reusability and scalability
- Zero backend infrastructure costs in v1

## 4. Technology Stack Architecture

### 4.1 Frontend Layer

| Concern              | Technology           | Rationale                                     |
|----------------------|----------------------|-----------------------------------------------|
| Framework            | Next.js 14 App Router| RSC, SSG, built-in API routes                 |
| Language             | TypeScript 5 (strict)| Type safety, intellisense, maintainability    |
| Styling              | Tailwind CSS 3       | Utility-first, no CSS conflicts in monorepo   |
| Animation            | Framer Motion 11     | Declarative, production-grade React animations|
| Form handling        | React Hook Form + Zod| Performant, schema-driven validation           |
| Icons                | Lucide React         | Consistent, tree-shakeable SVG icons          |
| Class merging        | clsx + tailwind-merge| Conflict-safe Tailwind class composition      |

### 4.2 Backend Layer (Serverless)

| Concern              | Technology           | Rationale                                     |
|----------------------|----------------------|-----------------------------------------------|
| API Routes           | Next.js Route Handlers| Co-located with frontend, no separate server |
| Email dispatch       | Nodemailer           | Mature, flexible SMTP library                 |
| Input validation     | Zod                  | Shared schemas with frontend                  |
| Rate limiting        | Custom in-memory     | No external service needed in v1              |

### 4.3 Infrastructure Layer

| Concern              | Technology           | Rationale                                     |
|----------------------|----------------------|-----------------------------------------------|
| Deployment           | Vercel               | Zero-config Next.js deployment, CDN, edge fn  |
| Monorepo             | Nx 18+               | Build caching, project graph, modularization  |
| Version control      | Git + GitHub         | Industry standard, Vercel GitHub integration  |
| CI/CD                | Vercel GitHub Actions| Auto-deploy on push to main                   |

## 7. Security Architecture

| Threat                      | Mitigation                                              |
|-----------------------------|---------------------------------------------------------|
| XSS via form input          | Zod .trim() + sanitize-html before email dispatch       |
| CSRF attacks                | SameSite cookie policy; no sessions in v1               |
| Spam / Bot submissions      | IP rate limiting (10 req/min), honeypot field (v2)      |
| SMTP credential exposure    | Environment variables; never committed to Git            |
| Dependency vulnerabilities  | npm audit in CI; Dependabot on GitHub                   |
| Sensitive data in logs      | No PII logged in API routes                             |
